## Git 版本控制

所有图片已添加到 Git 仓库，配置在 `server-nestjs/.gitignore` 中：

```gitignore
# 上传文件（忽略用户上传，保留默认资源）
/uploads/*
!/uploads/images/
!/uploads/images/**
!/uploads/photos/
!/uploads/photos/**
!/uploads/idcard/
!/uploads/idcard/**
!/uploads/audio/
!/uploads/audio/**
```

这样配置确保：
- ✅ 初始资源图片会被提交到 Git
- ✅ 用户后续上传的图片不会被提交
- ✅ 团队成员克隆项目后即可看到完整的初始数据

## 验证图片

运行验证脚本检查所有图片是否存在且有效：

```bash
bash server-nestjs/scripts/verify-images.sh
```

## 重新下载图片

如果需要重新下载所有图片：

```bash
bash server-nestjs/scripts/download-images.sh
```

## 在应用中使用

### 管理后台（Web）

图片通过 NestJS 静态文件服务访问：

```
http://localhost:3000/uploads/images/city-hangzhou.jpg
http://localhost:3000/uploads/photos/westlake-duanqiao-1.jpg
```

### 移动端（Flutter）

图片 URL 存储在数据库中，通过 API 返回：

```dart
// 城市封面图
Image.network('${apiBaseUrl}/uploads/images/city-hangzhou.jpg')

// 探索照片
Image.network('${apiBaseUrl}/uploads/photos/westlake-duanqiao-1.jpg')
```

## 注意事项

1. **初始数据以 seed.ts 为准**：所有图片路径和数据结构都与 `server-nestjs/prisma/seed.ts` 保持一致
2. **图片格式**：所有图片均为 JPEG 格式，兼容性好，文件大小适中
3. **响应式设计**：提供了原图和缩略图两种尺寸，适配不同场景
4. **占位图说明**：当前使用的是占位图，生产环境建议替换为实际的城市和景点照片
