# WiFi 代理检查快速参考

## 🔍 如何检查 WiFi 是否有代理

### iPhone/iPad

```
设置 → Wi-Fi → 点击已连接 WiFi 的 (i) → 滚动到「HTTP 代理」
```

### 三种状态

| 状态 | 说明 | 是否正常 |
|------|------|---------|
| ○ **关闭** | 没有代理 | ✅ 正常 |
| ● **手动** | 配置了代理服务器 | ⚠️ 需要检查 |
| ● **自动** | 使用 PAC 自动配置 | ⚠️ 需要检查 |

## 🚨 常见代理地址

| 地址 | 说明 |
|------|------|
| `127.0.0.1:8888` | Charles 抓包工具 |
| `127.0.0.1:9090` | Proxyman 抓包工具 |
| `192.168.x.x:xxxx` | 局域网代理 |
| `proxy.company.com:8080` | 公司代理 |

## ⚡ 快速解决方案

### 方案 1：关闭代理（最简单）
```
设置 → Wi-Fi → (i) → HTTP 代理 → 选择「关闭」
```

### 方案 2：换个 WiFi（最快）
```
设置 → Wi-Fi → 连接其他 WiFi（家里的、手机热点等）
```

### 方案 3：用 4G/5G（临时）
```
关闭 WiFi，使用蜂窝数据
```

## 💡 记住这一点

**每个 WiFi 的代理配置是独立的！**

- WiFi A 有代理 ≠ WiFi B 有代理
- 公司 WiFi 有代理 ≠ 家里 WiFi 有代理
- 切换 WiFi 是最简单的解决方案

## 🔧 如何关闭代理

1. 打开「设置」
2. 点击「Wi-Fi」
3. 点击已连接 WiFi 的 **(i)**
4. 滚动到「HTTP 代理」
5. 选择「○ 关闭」
6. 点击右上角「完成」或返回

## ✅ 验证是否修复

运行 App 后查看日志：

**✅ 成功（无代理问题）：**
```
[DIO] statusCode: 200
[DIO] data: {...}
```

**❌ 失败（代理问题）：**
```
[ERROR] Connection reset by peer
[ERROR] errno = 54
```
